# AI System Lifecycle

## Purpose
Define lifecycle phases, deliverables, and evidence required for audit readiness and EU AI Act alignment.

## Lifecycle phases and evidence
| Phase | Key deliverables | Required evidence | EU AI Act relevance (technical + organizational) |
| --- | --- | --- | --- |
| Design | System boundaries, risk ownership model, requirements baseline | Approved requirements, boundary statement, risk ownership record | Risk management planning, governance structure |
| Build | Control specifications, data handling rules, logging requirements | Control mappings, test scenarios, policy drafts | Technical safeguards and data governance |
| Deploy | Environment approvals, deployment checklist, access controls | Change approvals, IAM review notes, encryption settings | Security, accountability, and traceability |
| Operate | Runbooks, monitoring thresholds, audit log retention | Operational dashboards, audit log samples | Ongoing monitoring and oversight |
| Monitor | KPI reporting, incident trend analysis | KPI dashboard exports, alert summaries | Performance monitoring, risk management feedback loop |
| Incident handling | Incident response playbooks, escalation logs | Incident reports, root cause analysis, lessons learned | Human oversight and accountability |
| Change management | Policy updates, configuration change records | Change tickets, approvals, audit trail | Lifecycle control and traceability |
| Decommission | Data retention disposition, access revocation | Decommission checklist, data disposal evidence | Data protection and accountability |

## Work reporting rule
Each lifecycle phase requires a work report entry per contributing role in `docs/work_reports/roles/`.
