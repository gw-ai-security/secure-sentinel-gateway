# Master Prompt - Secure Sentinel AI Gateway (SSG)

## Purpose
Drive a documentation-first, enterprise-grade AI governance project using a coordinated, role-based team and a mandatory Knowledge Base.

## Operating rules
- No application code or implementation details.
- Follow phase order and do not skip documentation or governance steps.
- Treat the Knowledge Base as the single source of truth.
- Every phase must update at least one Knowledge Base file.
- Every decision must be traceable to the Knowledge Base.

## Roles and responsibilities
Use the Codex Skills in `docs/skills/` and keep roles strictly separated. Required roles include:
- Team: Team Orchestration, Traceability and Quality, Portfolio Storytelling.
- Roles: Requirements Engineer, AI Architect, AI Security Architect, Data Engineer, DevOps Engineer (AWS), Compliance Officer (EU AI Act Technical), Compliance Officer (EU AI Act Organizational), AI Tutor Technical Writer, Knowledge Base Curator, AI Program and Governance Lead.
- Methods: Threat Modeling.

## Phase gates
- Phase 0: Project structure and skills prepared.
- Phase 0B: Knowledge Base initialized and populated.
- Phase 1: Requirements completed and linked to Knowledge Base.
- Phase 1B: Recruiter-friendly README completed with Knowledge Base emphasis.

## Evidence expectations
- Maintain traceability from requirements to controls, threats, and compliance mappings.
- Keep audit evidence lists current.
- Record all governance decisions in the Knowledge Base.
